# Contributing to Truly Social

This project is an open inquiry. It grows through people who have a philosophical claim about human connection and want to make it tangible.

You do not need to be a developer to contribute. The most valuable contributions start as ideas.

---

## What is a kernel

A kernel is a small, working proof. It takes one claim about what genuine social connection requires and makes it felt through interaction - not argued, not explained. Felt.

A good kernel:
- Is built on a specific philosophical claim, not a general concern
- Creates friction that reveals something the person didn't notice before
- Leaves the person who uses it slightly different than before
- Runs in a browser with no install, no build step, no account

A kernel is not:
- A product
- A solution
- A statement about what platforms should do

It is an experiment. It asks "what if this one thing were required?" and lets you feel the answer.

---

## How to propose a kernel

Open an issue with:

**The claim** - one sentence. What does genuine connection require that current platforms do not provide?

**The thinker** - whose work grounds this? Not required, but encouraged.

**The friction** - what would the interaction feel like? What would be required before you could proceed?

**What it measures** - what would change in someone who used it?

That is enough to start a conversation. Someone will build it, or you will.

---

## How to build a kernel

Each kernel lives in its own folder:

kernels/
your-kernel-name/
README.md
index.html
NOTES.md

**README.md** - the philosophical brief. The claim, the thinker, what the experiment measures, what it cannot prove. Use the witness kernel as a template.

**index.html** - the working prototype. Single file. No framework required. No build step. Should run by opening in any browser.

**NOTES.md** - start this empty. It is where observations go over time - what people felt, what broke, what surprised you.

---

## The format constraint is intentional

Single file. No dependencies. Runs anywhere.

This keeps every kernel accessible to anyone with a browser. It also keeps the focus on the idea rather than the implementation. If your kernel needs a backend, a database, or a login - it is probably trying to do too much.

---

## What we are not building

This is not a social network. It is not a startup. No kernel should collect data, require an account, or simulate a platform. Each one is a standalone thought experiment that runs locally.
